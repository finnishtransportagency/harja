Kuvaus

Harja -järjestelmän julkinen API

API on jaettu kuuteen pääosioon:

  1. urakat:
    • Urakoiden perustietojen haku joko urakoitsijan id:llä tai yksittäisen urakan id:llä
    • Urakan seurantatietojen kirjaus: toteumat, tarkastukset, turvallisuuspoikkeamat, päivystäjätiedot, laatupoikkeamat sekä tietyöilmoitukset
    • Urakan Tieliikennekeskuksen tekemien ilmoitusten haku
  2. varusteet:
    • Varustetietojen tietolajikuvausten haku Tierekisteristä
    • Varustetietojen haku Tierekisteristä
    • Varustetietojen lisäys, poisto ja päivitys Tierekisteriin.
  3. seuranta:
    • Reaaliaikaisten seurantatietojen, kuten esim. aura-auton liikkeiden kirjaus
  4. ilmoitukset:
    • Tieliikennekeskuksen urakalle tekemien ilmoitusten haku
    • Kuittausten eli toimenpiteiden tekeminen ilmoituksille

  5. tieluvat:
    • Tielupien haku
  6. päivystäjätiedot:
    • Päivystäjätietojen haku

Tekninen kuvaus

Harja -järjestelmän rajapinta on toteutettu REST-API:na, jossa tieto liikkuu JSON-formaatissa.

Tietoliikenne Harjan palvelimille kulkee Liikenneviraston keskitetyn gatewayn kautta (Oracle Enterprice Gateway). Siirtoprotokollana käytetään HTTPS. Käyttäjän tunnistaminen tehdään käyttäen HTTP Basic autentikaatiota tunnus-salasana -parina. Liikennevirasto hallinnoi järjestelmä-/urakoitsijakohtaisia tunnuksia.

Kutsut voi lähettää gzipillä pakattuina merkitsemällä content encodingin gzipiksi.

Huomioitavaa:

  • Kaikki ajat raportoidaan Suomen aikavyöhykkeessä GMT+2/3. Aikaleimat täytyy siis raportoida formaatissa: "2016-11-03T03:43:14+02:00"
  • Kaikki toteumien tehtävien ja materiaalien määrät raportoidaan Harjan urakkahaun palauttamassa yksikössä. Harja ei tee muunnoksia raportoiduille yksiköille.

Sijaintitiedot

Kaikki koordinaattisijainnit ilmoitetaan Harjaan EUREF-FIN koordinaatistossa. Koordinaattipisteiden lisäksi vastaanotetaan suunta joka on aseteluku väliltä: 0-360.
